# Laju Go ÔÇö Agent Instructions

High-performance SaaS boilerplate: Go Fiber + Svelte 5 + Inertia.js + SQLite + templ.

## Architecture

Single-entry Go app at `cmd/laju-go/main.go`.

```
routes/web.go     Ôćĺ app/handlers/     Ôćĺ app/services/     Ôćĺ app/queries/     Ôćĺ SQLite
                middlewares/        session/          (sqlc generated)
```

- **Handlers**: Parse requests, call services, return responses. No business logic.
- **Services**: Business logic, auth flows, external APIs.
- **Queries**: `app/queries/` is **generated by sqlc** from `queries/*.sql`. Never edit manually.
- **Templates**: `templates/*.templ` ÔÇö type-safe HTML components via [templ](https://templ.guide/). Generated `*_templ.go` files must be committed.
- **Frontend**: Svelte 5 + Inertia.js. Entry at `frontend/src/main.ts`. Built to `dist/`.

> **goose vs Goose AI**: Semua command migrasi pake `go run github.com/pressly/goose/v3/cmd/goose@latest` (bukan binary `goose`) supaya tidak ambigu dengan [Goose AI agent](https://block.github.io/goose/).

**Build order matters**: `vite build` must run before `go build` in production because the Go binary reads `dist/.vite/manifest.json`.

**Windows users**: Scripts via `npm run` work natively. For `make`, use **WSL** (`wsl make build`).

## Templ Workflow

- Edit `.templ` files Ôćĺ run `templ generate` Ôćĺ commit both `.templ` and `*_templ.go`.
- **Air does NOT watch `.templ` files** by default. Add `"templ"` to `.air.toml` `include_ext` or regenerate manually.
- Rendering: `templates.ComponentName(args...).Render(ctx, writer)` instead of `c.Render("name", data)`.

## Inertia.js Pattern

- Initial page load Ôćĺ server renders `templates.InertiaPage` HTML shell with JSON page data.
- Subsequent navigation Ôćĺ XHR with `X-Inertia: true` header Ôćĺ server returns JSON `{component, props, url}`.
- Handlers use `inertiaService.Render(c, "ComponentName", fiber.Map{...})` ÔÇö never render HTML directly for Inertia pages.

## Database & Migrations

- SQLite via modernc.org/sqlite (pure-Go, CGO-free).
- Migrations run **automatically on startup** via Goose.
- Write SQL in `queries/*.sql`, then `npm run db:generate` to create typed Go code.
- `sqlc.yaml` configures the generation. Schema source is `migrations/`.

### Migration Convention ÔÇö One Table Per File

Setiap file migrasi harus berisi **satu tabel saja**. Jangan menggabungkan beberapa tabel dalam satu file migrasi.

Ôťů **Benar:**

```
migrations/
ÔöťÔöÇÔöÇ 0001_create_users_table.sql       -- hanya CREATE TABLE users
ÔöťÔöÇÔöÇ 0002_create_sessions_table.sql     -- hanya CREATE TABLE sessions
ÔööÔöÇÔöÇ 0003_create_password_resets_table.sql -- hanya CREATE TABLE password_resets
```

ÔŁî **Salah (jangan lakukan ini):**

```sql
-- 0001_initial.sql ÔÇö ÔŁî multiple tables in one file
CREATE TABLE users (...);
CREATE TABLE sessions (...);
CREATE TABLE password_resets (...);
```

Alasan:

1. **Isolasi migrasi** ÔÇö Jika migrasi `sessions` gagal, tabel `users` tetap ter-migrasi. Dengan satu file besar, semua gagal.
2. **Rollback granular** ÔÇö `goose down` bisa rollback tabel spesifik.
3. **History jelas** ÔÇö Setiap tabel punya timestamp migrasi sendiri.
4. **sqlc schema source** ÔÇö sqlc membaca `migrations/` untuk schema. File terpisah = lebih mudah di-debug.

Setiap file migrasi WAJIB memiliki `-- +goose Up` dan `-- +goose Down` section.

### Database driver

`github.com/mattn/go-sqlite3` (CGO-based). Requires CGO for cross-compilation. Cross-compile ke Linux dari macOS menggunakan `brew install zig` lalu `make build-linux`.

## Design Standards

Setiap halaman yang digenerate (SSR `.templ` maupun Inertia Svelte) harus:

1. **Visual Hierarchy** ÔÇö Focal point jelas, ukuran/weight/kontras mengarahkan mata
2. **Whitespace lega** ÔÇö Jangan sesak. Padding 16-24px minimum,ÚŚ┤ŔĚŁ antar section 32-48px
3. **Typography** ÔÇö Heading bold/berat, body clean/sans-serif. Tracking (letter-spacing) longgar untuk headline
4. **Color** ÔÇö 60-30-10 rule. Netral dominan, aksen minimal. Hindari warna jenuh kecuali diminta
5. **Micro-interactions** ÔÇö Hover state, transisi smooth, shadow halus. Gak ada elemen mati
6. **Mobile-first** ÔÇö Semua halaman harus responsif. Test di viewport 375px, 768px, 1440px

Kalau ragu dengan visual, minta screenshot via agent_browser ÔÇö saya review dan fix.

## Svelte 5 Rules

- **Never use `$effect` for simple initial state or UI sync.** Initialize values directly in `$state(...)`, or compute with `$derived(...)`. `$effect` is only for side effects that interact with non-reactive APIs (e.g. `document.title`, `setInterval`).
- Components receive Inertia page data as props via `$props()`. Props are available at component instantiation ÔÇö initialize derived form state at the top level, not in `$effect`/`onMount`.

#### Anti-patterns ÔÇö Jangan

```svelte
<!-- ÔŁî BAD: $effect for derived state -->
<script>
  let { items } = $props();
  let filtered = $state([]);
  $effect(() => { filtered = items.filter(...); });
  // Ôťů Ganti: let filtered = $derived(items.filter(...));
</script>

<!-- ÔŁî BAD: $effect for initialization -->
<script>
  let { initial } = $props();
  let count = $state(0);
  $effect(() => { count = initial; });
  // Ôťů Ganti: let count = $state(initial ?? 0);
</script>
```

#### $effect hanya untuk

```svelte
<!-- Ôťů OK: Side effect ke luar sistem -->
<script>
  let { user } = $props();
  $effect(() => { document.title = user.name; });
  $effect(() => { localStorage.setItem('theme', theme); });
</script>
```

## Conventions

- **Handlers must NOT call queries directly.** All database access goes through services. Handler Ôćĺ Service Ôćĺ Query. No shortcut from handler to query.
- **POST/PUT handlers that redirect**: Use `c.Redirect(path, fiber.StatusSeeOther)` (303). Inertia does not follow 302 correctly for form submissions ÔÇö it needs 303 to change POST/PUT to GET on redirect.
- **PUT/PATCH**: Return JSON for `fetch()` calls, redirect for `router.put()` calls. If redirecting, always 303.
- Sessions are database-backed (SQLite table). Auth middleware checks `session.Store`.
- CSRF: Axios (Inertia's HTTP client) auto-sends cookie `XSRF-TOKEN` as header `X-XSRF-TOKEN`. Cookie is set by CSRF middleware on GET responses (`HTTPOnly: false`). CSRF middleware only on `/app/*` routes.
- `fiber.Map` for untyped response data. Use typed structs for service boundaries.

## Environment

Copy `.env.example` Ôćĺ `.env`. Minimum required:

- Google OAuth and SMTP are optional.

## Gotchas

- `.vite-port` file is written by Vite and read by Go. If stale, `rm .vite-port` and restart Vite.
- `go.sum` is gitignored. Run `go mod tidy` locally if needed.
- `dist/` is gitignored except `.gitkeep`. Build artifacts are not committed.
- Cross-compile for Linux: `make build-linux` (requires `brew install zig` for CGO cross-compile via `zig cc`).
- Air's `include_ext` does not include `.templ` ÔÇö regenerate templ components manually when editing templates.

## Testing Strategy

| Approach | For | How |
|----------|-----|-----|
| Go unit/integration | Services, queries, handlers, cache | `go test ./...` ÔÇö in-memory SQLite, no external dependencies |
| E2E / browser | Auth flows, form submission, page load, visual regression | `agent_browser` via pi ÔÇö real browser, no mock |

### Agent Browser Testing

Gunakan `agent_browser` untuk E2E testing selama development:

```
agent_browser args=["open", "http://localhost:8080/register"]
Ôćĺ snapshot, fill form, submit
Ôćĺ snapshot verify redirect ke /app

agent_browser args=["open", "http://localhost:8080/app/profile"]
Ôćĺ snapshot (harus redirect ke /login karena belum login)
```

**Tidak perlu** Cypress, Playwright, vitest, atau mock Inertia.
